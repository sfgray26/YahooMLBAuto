# Railway Deployment Guide

## Quick Start

### 1. Create Railway Project

```bash
# Install Railway CLI if not already installed
npm install -g @railway/cli

# Login
railway login

# Create new project (or link to existing)
railway init --name cbb-edge-analyzer
```

Or via web UI:
1. Go to https://railway.app/dashboard
2. Click "New Project"
3. Select "Deploy from GitHub repo"
4. Choose `sfgray26/YahooMLBAuto`

### 2. Add PostgreSQL Plugin

```bash
railway add --database postgres
```

Or via web:
1. Click "New" → "Database" → "Add PostgreSQL"
2. Railway auto-generates `DATABASE_URL`

### 3. Add Redis Plugin

```bash
railway add --database redis
```

Or via web:
1. Click "New" → "Database" → "Add Redis"
2. Railway auto-generates `REDIS_URL`

### 4. Configure Services

Railway will detect `railway.toml` and create 5 services automatically:
- `api` - HTTP API (public)
- `worker` - Background job processor
- `scheduler` - Cron job runner (every 15 min)
- `scheduler-daily-ingest` - Twice-daily game log + derived stat refresh
- `db-migrate` - One-off migration job for deploys

### 5. Set Environment Variables

These are auto-generated by the plugins, but verify they exist:

```bash
railway variables

# Should show:
# DATABASE_URL=postgresql://...
# REDIS_URL=redis://...
```

Optional variables:
```bash
railway variables set LOG_LEVEL=info
railway variables set WORKER_CONCURRENCY=5
railway variables set ALLOW_MOCK_VALUATIONS=false
```

### 6. Deploy

```bash
# Deploy all services
railway up

# Or deploy specific service
railway up --service api
railway up --service worker
```

## Troubleshooting

### Build Fails

```bash
# Check build logs
railway logs --service api

# Common issues:
# 1. Missing pnpm-lock.yaml - Run `pnpm install` locally and commit
# 2. Type errors - Run `pnpm typecheck` locally
```

### Database Connection Fails

```bash
# Verify DATABASE_URL is set
railway variables get DATABASE_URL

# Check if migration ran
railway run --service api -- npx prisma migrate status
```

### Worker Not Processing Jobs

```bash
# Check worker logs
railway logs --service worker

# Verify Redis connection
railway run --service worker -- node -e "console.log(process.env.REDIS_URL)"
```

## Local Development with Railway

```bash
# Link local project to Railway
railway link

# Run local server with Railway env
railway run -- pnpm api:dev

# Run worker with Railway env
railway run -- pnpm worker:dev
```

## Scaling

### Horizontal (More instances)

```bash
# Increase API replicas
railway scale --service api --replicas 2

# Note: Worker should typically stay at 1 (concurrency handled by BullMQ)
```

### Vertical (More resources)

Edit `railway.toml`:
```toml
[services.api.deploy.resources]
memory = "1Gi"  # Increase from 512Mi
```

Then redeploy:
```bash
railway up
```

## Domain & SSL

Railway auto-provisions SSL for custom domains.

```bash
# Add custom domain
railway domain add your-domain.com

# DNS will be provided - add CNAME record
```

## Database Migrations

Migrations run automatically on deploy via `railway.toml` (you can add a deploy command).

Manual migration:
```bash
railway run -- npx prisma migrate deploy
```

## Backup Strategy

Railway PostgreSQL has automatic backups (daily).

Manual backup:
```bash
railway run -- pg_dump $DATABASE_URL > backup.sql
```

## Cleanup

To delete everything:
```bash
railway down --yes
```
